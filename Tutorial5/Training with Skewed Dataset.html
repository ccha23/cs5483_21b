2.1. SetupÂ¶

In this notebook, we will train classifiers properly on the skewed dataset for detecting microcalcifications in mammograms.

In particular, we will use the meta classifier ThresholdSelector and the filter SMOTE Synthetic Minority Over-sampling Technique, which needs to be installed as additional packages in WEKA:

jvm.start(packages=True)  # support custom packages

import weka.core.packages as packages

pkgs = ["thresholdSelector", "SMOTE"]

for item in packages.all_packages():
    if item.name in pkgs:
        print(item.name + " " + item.url)

You may install the packages using Weka package manager instead of downloading the zip files. To install them in python-weka-wrapper, run the following code:

for pkg in pkgs:
    if not packages.is_installed(pkg):
        print(f"Installing {pkg}...")
        packages.install_package(pkg)
    else:
        print(f"Skipping {pkg}, already installed. ")
else:
    print("Done.")

By default, these packages are installed under your home directory ~/wekafiles/packages/:

!ls ~/wekafiles/packages

For the packages to take effect, you must restart the kernel (Kernel -> Restart). Note that running jvm.stop() followed by jvm.start(packages=True) will not work because javabridge currently does not support restarting a virtual machine.

After restarting the kernel, check that the packages have been successfully installed by running the following code:

2.2. Threshold SelectorÂ¶

The meta classifier ThresholdSelector uses the threshold-moving technique to optimize a performance measure you specify, which can be the precision, recall, \(F\)-score, etc. See an explanation of threshold moving technique here.

The following shows how to maximize recall:

tsc = SingleClassifierEnhancer(
    classname="weka.classifiers.meta.ThresholdSelector",
    options=['-M', 'RECALL'])
tsc.options=['-M', 'RECALL']
tsc.classifier = clf

evl = Evaluation(data)
evl.crossvalidate_model(tsc, data, 10, Random(1))

print(f"maximum recall: {evl.recall(pos_class):.3g}")

The maximum recall is 100% as expected by setting the threshold to 1.

2.3. Cost-sensitive ClassifierÂ¶

We can build a classifier to maximize certain cost. Weka provides a convenient interface for the cost/benefit analysis:

  1. In the explorer interface, train J48 on the mammography dataset with 10-fold cross-validation.
  2. Right click on the result in the result list.
  3. Choose Cost/Benefit analysis and 1 as the positive class value.
  4. Specify the cost matrix:
    \begin{align} \begin{bmatrix} \text{cost}_\text{TP} & \text{cost}_\text{FN}\\ \text{cost}_\text{FP} & \text{cost}_\text{TN}\end{bmatrix}. \end{align}
  5. Click `Minimize Cost/Benefit` to minimize the cost:
    \begin{align} \text{cost}_\text{TP} \text{TP} + \text{cost}_\text{FN} \text{FN} + \text{cost}_\text{FP} \text{FP} + \text{cost}_\text{TN} \text{TN}. \end{align}

csc = SingleClassifierEnhancer(
    classname="weka.classifiers.meta.CostSensitiveClassifier",
    options=[
        "-cost-matrix",

precision = evl.precision(pos_class)
print(f"maximum precision: {precision:.3g}")

2.4. SMOTEÂ¶

Synthetic Minority Over-sampling TEchnique (SMOTE) is a filter that up-samples the minority class. Instead of creating duplicates of the same instance, it creates new samples as convex combinations of existing ones. See a more detailed explanation of SMOTE here.
